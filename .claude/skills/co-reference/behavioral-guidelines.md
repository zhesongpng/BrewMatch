# Behavioral Guidelines for Framework Discussions

## General Rules

- **Ground in specs** — use the specific spec files (care-spec, eatp-spec, co-spec, coc-spec) as authoritative source
- **Be honest about limitations** — every framework explicitly acknowledges what it does not solve
- **Distinguish normative from ontological** — the Dual Plane separation is a useful framing, not a discovered truth
- **Connect frameworks as a quartet** — CARE (philosophy), EATP (protocol), CO (methodology), PACT (governance) are peers, not competitors

## CARE-Specific

- Emphasize Trust Plane is permanently human — normative choice, not technical limitation
- Distinguish the six competencies as current AI limitations, not principled impossibilities
- CARE is vendor-agnostic: works above any execution tool

## EATP-Specific

- Be precise about terminology — Genesis Record, Capability Attestation, Delegation Record, Constraint Envelope, Audit Anchor have specific meanings
- **Distinguish traceability from accountability** — EATP's most important distinction
- Connect to CARE — EATP operationalizes CARE's governance philosophy

## CO-Specific

- Be domain-agnostic — CO applies to any domain; use neutral language
- Reference COC as proof — COC is the proven domain application
- Be honest about maturity — CO is newly formalized from COC; COC is battle-tested

## COC-Specific

- Ground in the three fault lines — Amnesia, Convention Drift, Security Blindness are the root problems
- Emphasize institutional knowledge — competitive advantage is context, not model capability
- Reference the Kailash implementation — five layers are implemented, not theoretical
