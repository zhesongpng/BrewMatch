## Findings (both fixed in-round)

- **F1 (LOW, output-encoding inconsistency) — FIXED.** User input (roaster/name)
  was escaped via `escape_markdown` in the bag-list card (lines 187/191) but NOT
  in the post-save `st.success` toast — `st.success` renders markdown, so a
  roaster like `*Sey*` would render italicized. Self-targeted, cosmetic, low
  severity, but inconsistent. Now escaped at all 3 display sites.

- **F2 (LOW, pre-existing UX bug, carried forward — FIXED.)** The "Origin =
  Other → specify origin" field was rendered conditionally INSIDE `st.form`.
  Inside a form, the selectbox value is not readable until submit, so the
  conditional field needed TWO submits to appear (and the first submit failed
  validation with an empty origin). Provenance: pre-existing — committed at SHA
  `6cd9611` (pre-session) with the identical conditional; carried forward in the
  B1.3 rewrite. Fixed by rendering the specify field unconditionally so "Other"
  works in one pass; it is read only when "Other" is selected.

## Notes (reviewed, not findings)

- `_grams_used` returns `0.0` and `_brews_left` uses a nominal 15 g dose. These
  are the honest session-state values (no brews logged against session bags) and

- Build/wire seams (`_load_active_bags` / `_save_bag` / `_grams_used`) localize
