## Findings

- **F1 (HIGH → fixed) — spec Now/Later framing.** The §4.5 edit ended with
  "_the rescaled recipe is wired into the saved brew record in a later step_,"
  a forward-reference that `spec-accuracy.md` Rule 2 blocks in spec content
  (split-state "Now / Later" / "to be wired" framing). A spec must describe what
  ships today, full stop; roadmap belongs in the todo. **Fixed:** rewrote the
  item to describe only shipped behavior — "the rescaling adjusts the displayed
  guidance only; the recipe stored with the brew record is unchanged." The
  split-state scan (`grep -niE 'phase1.*phase2|to be wired|later step|...' specs/`)
  is now clean.

## Adversarial probes that came back clean (not findings)

- **Rounding vs the ratio invariant.** `_scale_recipe` rounds water and each
  pour to 0.1 g independently while keeping the original `ratio`. Theoretically
  rounding could push `|ratio − water/dose|` past the model's 0.1 tolerance and
  raise on a _legitimate_ dose. Brute-forced every real recipe × every 0.5 g
  step from 12–25 g (1242 builds): **0 spurious rejections** and 0 in-range
  rejections of any kind. Real recipes carry `|ratio − water/dose| ≤ 0.033`, far
  inside the budget; the safety-net try/except is correct but rarely exercised
  in practice.
- **Degenerate number_input bounds.** `low=min(12, dose)`, `high=max(25, dose)`
  — `low < high` for every Recipe-valid dose (12–35 g), so the widget never gets
  `min == max`. Default `value = recipe.dose_g` ⇒ screen opens unchanged, no
  surprise rescale on load.
- **Pour floor on downscale.** A pour dropping below 10 g raises via
  `PourStep.__post_init__` inside `_scale_recipe`; same try/except fallback as
  the water floor. Covered by the same rejection path.

## Known limitation (bounded, tracked — not a B1.5 defect)

- **Dialed-in dose is not persisted yet.** B1.5 is display-only by design: a
  user who sets 18 g sees rescaled guidance, but the saved brew record keeps the
  recipe's original dose. This is the explicit B1.5 → B1.6 split (B1.6 wires the
  scaled recipe + `actual_dose_g` + bag link into the brew record and the
  running-low count). Documented accurately in §4.5 and tracked as the next
  todo. Logged as journal RISK 0028 so it is not forgotten if B1.6 slips.
