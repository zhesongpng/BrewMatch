## Findings

- **F1 (MED → fixed) — no-bag guard was untested.** The B1.6 guard that drops a
  stale `current_bag_id` on the one-off path lived inline in `_submit_feedback`
  (Streamlit-bound, never exercised by any test). A regression there — a no-bag
  brew silently inheriting a previously-picked bag and decrementing the wrong
  bag's count — would have shipped undetected. **Fixed:** extracted the decision
  into a pure `_resolve_bag_id(current_bean, current_bag_id)` helper and added
  `TestResolveBagId` (3 cases: bean+bag → link; no bean → drop stale id; bean
  but no bag → none). Mutation M4 confirms the test catches the regression.
